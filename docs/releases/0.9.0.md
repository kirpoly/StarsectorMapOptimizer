# Отчёт о выпуске 0.9.0

Дата: 2026-07-18.

`0.9.0` добавляет поддерживаемый запуск StarsectorPrepatcher вместе с **Faster Rendering**. Ранее
простое добавление agent в `fr.vmparams` оставляло typed runtime hooks в loader'е javaagent, тогда
как Faster Rendering child-load'ил игровые `com.fs.*` классы отдельным system loader. Первый
deferred-вызов hook с `CampaignEngine` или другим игровым типом мог завершиться
`loader constraint violation`/`LinkageError`, в частности при генерации новой кампании.

## Исправление classloader topology

Control plane и typed runtime теперь разделены:

- `com.starsector.prepatcher.agent.*` остаётся в loader'е javaagent и управляет configuration,
  structural matching, ASM и регистрацией transformer;
- payload `com.fs.starfarer.api.StarsectorPrepatcher*` читается из agent JAR как classfile bytes;
- `RuntimeInstaller` получает `com.fs.starfarer.api.Global` через system loader и определяет все
  48 payload-классов через `MethodHandles.privateLookupIn(...).defineClass(...)`;
- transformer регистрируется только после полной установки и configuration payload;
- target, принадлежащий другому loader'у, остаётся неизменённым со статусом `SKIPPED_LOADER` вместо
  позднего `LinkageError`.

`sound.Sound` Faster Rendering оставляет в parent loader. Его отключённый по умолчанию
`patch.startupLogAggregation` сохраняет structurally-verified inline путь без typed helper call.

## Установка

Installer и uninstaller поддерживают три явные цели:

```text
Vanilla | FasterRendering | Both
```

Для Faster Rendering используется команда:

```bat
install-agent.bat -Target FasterRendering
```

Installer понимает root `vmparams` и `starsector-core/fr.vmparams`, сначала проверяет все выбранные
файлы, создаёт timestamped backup и идемпотентно размещает Prepatcher после остальных
`-javaagent`. При совместном использовании рекомендуемый порядок — Telemetry, затем Prepatcher.

## Проверка

Полный release gate на Starsector `0.98a-RC8` подтвердил:

- 27 target-классов и 1753 concrete methods в structural/negative/idempotency suite;
- runtime, lifecycle/GC, loading/save и hyperspace regressions;
- 4 hyperspace targets и 123 verified methods;
- настоящий `com.genir.renderer.loaders.AppClassLoader` из поставляемого `fr.jar`;
- 48 payload-классов в FR system/game loader и control plane в отдельном
  `AppClassLoader$JavaAgentLoader`;
- `APPLIED` для `CampaignEngine` и реального caller `IntelManager.commRelaySystemIndex`;
- разрешение typed descriptor `commRelayCandidateSystems(CampaignEngine, Vector2f, float, float)`;
- process-level порядок Telemetry → Prepatcher на bundled Java 17.0.10.

Последняя проверка подтверждает только совместное выполнение premain и classloader/linkage topology;
она не является функциональным тестом сбора данных сторонним telemetry-модом.

## Остаточные ограничения

Runtime payload пока разрешает `PrepatcherConfig` и `PrepatcherLog` через проверенный fallback
Faster Rendering из system `AppClassLoader` в `JavaAgentLoader`. Изменение resolver в будущей версии
FR должно fail-open остановить установку до регистрации transformer. Полностью JDK-only config/log
boundary остаётся следующим усилением архитектуры.

Публичный API `com.starsector.prepatcher.api` в `0.9.0` не выпущен. Формат сохранений не меняется:
runtime-кэши и scratch buffers существуют только в памяти текущей JVM.

Полный список пользовательских изменений находится в [`CHANGELOG.md`](../../CHANGELOG.md), правила
совместимости — в [`COMPATIBILITY.md`](../COMPATIBILITY.md), а обязательная игровая матрица — в
[`VALIDATION.md`](../VALIDATION.md).
