# Отчёт о выпуске 0.9.3

Дата: 2026-07-18.

`0.9.3` закрывает второй массовый vanilla-путь `Market.advance(float)`, обнаруженный реальной
observer-сессией: `planetConditionMarketOnly` рынки обновляются из `BaseCampaignEntity.advance()` и
раньше полностью обходили central remote-market scheduler. Выпуск также исправляет attribution
observer-а, чтобы этот известный engine path больше не выглядел как прямые вызовы неизвестных модов.

## Scheduler planet-condition-only рынков

`patch.planetConditionMarketScheduler` заменяет только один структурно подтверждённый call site:

```text
BaseCampaignEntity.advance(float)
  -> market.isPlanetConditionMarketOnly()
  -> MarketAPI.advance(amount)
```

Удалённые condition-only markets получают stable phase с cadence `4` по умолчанию. Первый tick
всегда выполняется немедленно, рынки в текущей location игрока остаются full-rate, а пропущенный
`amount` суммируется и ограничивается frame/day caps. Scheduler поддерживает тот же opt-out key:

```text
$starsectorPrepatcher_fullRateMarket=true
```

State хранится в отдельной identity map и не объединяется с central scheduler. Это сохраняет два
отдельных вызова, если нестандартный market одновременно встречается в обоих vanilla-путях. Перед
save pending time обоих scheduler-ов flush'ится раздельно обычными `MarketAPI.advance()` calls;
state transient и очищается на campaign generation boundary.

Если central scheduler выключен, transformer добавляет только необходимый frame-clock boundary и не
меняет central market-loop. Reentrant, duplicate-in-frame, invalid amount, отсутствующий frame
context и ошибка policy/state приводят к локальному immediate fail-open.

Патч performance-first: callback count удалённых condition-only markets уменьшается. Суммарное
elapsed time сохраняется, но plugins, использующие число callbacks вместо `amount`, могут изменить
поведение. Safe profile оставляет патч выключенным.

## Исправления observer-а

Level-1 observer теперь:

- отдельно классифицирует scheduled, fallback, save-flush и immediate planet-condition origins;
- не относит массовый известный vanilla path к `unknown_entries`;
- записывает metadata успешно трансформированного mod call site в `call-sites.csv` сразу после
  bytecode verification, даже если site ни разу не исполнился;
- обновляет лимит unknown stack samples каждый report interval и атомарно отсоединяет signature map,
  поэтому первые частые вызовы процесса не скрывают поздние reflection/MethodHandle paths;
- использует schema 2 `session.json` с `sessionOrigin`;
- маркирует каталоги startup/FR validation smoke отдельным префиксом, чтобы их нельзя было принять за
  игровую телеметрию.

Прямые mod calls по-прежнему синхронны и немедленны. Их receiver, `amount`, multiplicity, thread,
порядок post-call-кода и exception propagation не меняются.

Новые поля `summary.csv`:

```text
planet_condition_scheduled_entries
planet_condition_fallback_entries
planet_condition_save_flush_entries
planet_condition_immediate_entries
```

## Совместимость и fail-open

Bridge в `BaseCampaignEntity` принимает только точную форму predicate/receiver/amount/interface call
и проверяет, что заменён ровно один site. Каждый блок остаётся независимым: неизвестная форма
planet path не отключает observer, central scheduler или другие патчи.

Scheduler не меняет публичный API, тип `MarketAPI`, market plugin implementations и save format.
Observer не удерживает ссылки на markets. Любая ошибка записи telemetry логируется, но original call
не подавляется.

## Проверка выпуска

Release gate подтверждает:

- documentation/version/link/checksum consistency;
- structural transform всех включённых targets, `Analyzer + BasicVerifier`, API invariance,
  idempotency, ownership и tamper-negative cases;
- независимые комбинации central scheduler on/off и planet scheduler on/off;
- exact amount conservation, stable phases, first tick, current-location full-rate, memory opt-out,
  save flush, lifecycle reset и reentrancy fallback;
- eager observer manifest, known-origin attribution и обновление unknown stack budget между двумя
  интервалами;
- lifecycle/GC exact cache inventory;
- loading/save, hyperspace и javaagent startup suites;
- 56 runtime payload classes в target-loader inventory.

Полный список switches находится в [`PATCHES.md`](../PATCHES.md), правила совместимости — в
[`COMPATIBILITY.md`](../COMPATIBILITY.md), игровая матрица — в [`VALIDATION.md`](../VALIDATION.md).
