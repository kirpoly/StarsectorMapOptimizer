# Отчёт о выпуске 0.9.5

Дата: 2026-07-18.

## Назначение

`0.9.5` объединяет параллельно разработанный no-op-callback patch с economy/commodity/temp-mod
гибридом `0.9.4`. Цель — уменьшить стоимость `Market.advance()` в сохранениях с тысячами markets и
industries, не добавляя второй javaagent и не перехватывая модовые callback overrides.

Полезная часть параллельной ветки была переработана. Вместо static helper-а с глобальным
`IdentityHashMap` состояние находится непосредственно в transformed `BaseIndustry`, а hot path
сводится к чтению двух boolean fields и decrement локального countdown. Condition callback skip не
вошёл в релиз: измеренный helper overhead превышал стоимость пустого inherited virtual callback.

## `patch.marketNoOpCallbacks`

### Что делает vanilla

Exact `BaseIndustry.advance(float)` на каждом market tick:

1. вызывает `isDisrupted()`;
2. при переходе из disrupted в normal вызывает `disruptionFinished()`;
3. обновляет `wasDisrupted`;
4. при `building=true` конвертирует amount в days, двигает construction progress и завершает
   строительство/upgrade.

У большинства полностью построенных и не disrupted industries этот путь не меняет состояние, но
выполняется для каждого market каждый frame.

### Direct dormant wrapper

Transformer:

- переименовывает исходное тело в private synthetic `spp$baseIndustryRawAdvance(float)`;
- устанавливает новый public wrapper `advance(float)`;
- добавляет два `private transient synthetic int` fields — state и audit countdown;
- добавляет branchless wake-prologue в `setDisrupted(float, boolean)`.

Первый вызов всегда выполняет полное исходное тело. После него exact inherited implementation
становится dormant, только если:

```text
building == false
wasDisrupted == false
runtime class не переопределяет advance/isDisrupted/getDisruptedKey
```

Следующий steady-state вызов не обращается к helper/runtime map и не делает allocation. Он проверяет
`building`, `wasDisrupted`, уменьшает countdown и возвращается. После
`market.noOpIndustryAuditFrames` выполняется полный retained vanilla call.

### Немедленные wake paths

Полный вызов выполняется без ожидания audit, если:

- началось строительство/upgrade (`building=true` читается каждый attempted call);
- disruption уже было обнаружено (`wasDisrupted=true`);
- вызван штатный `BaseIndustry.setDisrupted(float, boolean)`;
- instance ещё не классифицирован или ранее переведён на vanilla fallback.

Custom runtime class, переопределяющий `advance(float)`, вообще не входит в base wrapper. Class,
который наследует `advance`, но переопределяет `isDisrupted()` или `getDisruptedKey()`, после первого
полного вызова получает permanent vanilla state. Eligibility кэшируется через `ClassValue`, поэтому
после классификации нет reflection на hot path и нет strong reference к classloader.

## Почему не перенесён condition callback skip

Параллельный вариант заменял каждый `MarketConditionPlugin.advance(float)` на helper, который через
`ClassValue` решал, является ли callback inherited no-op. В изолированном benchmark пустой virtual
callback стоил меньше nanosecond, тогда как helper/classification path — несколько nanoseconds.
Даже корректный skip увеличивал суммарное время `Market.advance()`.

Поэтому `0.9.5`:

- не меняет condition call site;
- не классифицирует `BaseMarketConditionPlugin`;
- не пропускает `LuddicPathCells`, `HostileActivity` или другие empty overrides;
- сохраняет callback cadence всех condition plugins буквально.

## Совместимость и намеренное изменение поведения

Патч уменьшает число вызовов только exact inherited dormant `BaseIndustry.advance()`. Это
осознанное исключение из общего compatibility-first правила сохранения callback cadence.

Сохраняются:

- public/protected API и descriptors;
- custom Industry `advance()` cadence и переданный `amount`;
- construction progress и completion;
- обычный disruption через `setDisrupted()`;
- порядок markets/industries;
- save format — оба поля transient;
- полный original method как fail-open source of truth.

Принятый риск: мод может наследовать `BaseIndustry.advance()`, но напрямую менять private memory key,
который читает inherited `isDisrupted()`, не вызывая `setDisrupted()`. Такое изменение станет видно
не позже следующего audit, по умолчанию через 120 attempted calls. Для exact cadence используется:

```properties
patch.marketNoOpCallbacks=false
```

Safe profile держит блок выключенным; default и aggressive включают.

При одновременном remote-market scheduling audit interval считается в доставленных market/industry
attempts, а не в глобальных engine frames. Поэтому `commodity.temporalAuditFrames`,
`market.structureAuditFrames` и `market.noOpIndustryAuditFrames` для рынка с cadence `4`/`8` могут
иметь примерно `4×`/`8×` wall-frame окно. Hot markets, safety wake и pre-save flush остаются
ограничителями; safe profile отключает scheduler paths.

## Structural guards

Патч применяется только если подтверждены:

- exact target `BaseIndustry` и один `advance(F)V`;
- один `isDisrupted()`, один `disruptionFinished()`;
- ожидаемые reads/writes `building` и `wasDisrupted`;
- exact `setDisrupted(FZ)V` contract;
- отсутствие partial/foreign `spp$marketNoOp*` members;
- template remapping без leaked template owner;
- retained raw method, wrapper wiring и wake postcondition;
- ownership marker, reparsing, public/protected API snapshot и ASM verification.

Любое несовпадение даёт локальный `SKIPPED_STRUCTURAL`; остальные patch blocks продолжают
рассматриваться.

## Проверка

Release suite включает:

- config/class-eligibility regression;
- structural positive, negative, partial, idempotency и ownership tests;
- actual-javaagent test с audit interval `2`;
- dormant skip и periodic full call;
- immediate wake через `setDisrupted`;
- full cadence для custom `advance`, `isDisrupted` и `getDisruptedKey`;
- проверку metadata synthetic fields/raw method;
- полный общий runtime, XStream/save, lifecycle/GC, loading/save, hyperspace и startup gate.

Изолированный benchmark прямого redesign на 2048 inherited industries показал медиану около
`5.22 ns/call` для vanilla и `2.42 ns/call` для patched hot path при audit `120`, то есть примерно
`2.16×` внутри этого микропути. Это не прогноз FPS: реальный эффект зависит от числа markets,
industries и доли `BaseIndustry.advance()` в профиле.

## Итог

`0.9.5` переносит только performance-positive часть параллельной разработки. Direct dormant
industry wrapper дополняет persistent Economy/Market snapshots, commodity active set и expiry-aware
temp-mod scheduler, не добавляя global per-instance maps и не замедляя condition callbacks.
