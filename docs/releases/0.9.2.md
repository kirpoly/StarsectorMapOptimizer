# Отчёт о выпуске 0.9.2

Дата: 2026-07-18.

`0.9.2` начинает отдельную серию оптимизаций экономики Starsector. Выпуск сокращает стоимость
центрального market-loop за счёт регулируемой частоты обновления удалённых рынков и одновременно
добавляет наблюдение за прямыми вызовами `MarketAPI.advance(float)` из модов. Следующие совместимые
итерации этого направления планируются как `0.9.3` и далее.

## Staggered scheduler рынков

`patch.remoteMarketScheduler` изменяет только вызов `MarketAPI.advance(float)` из центрального
`Economy.advance(float)`. Для удалённых рынков он накапливает исходный `amount` и применяет его на
стабильных фазах:

- обычные удалённые рынки в default/aggressive profile обновляются примерно раз в четыре кадра;
- скрытые рынки используют отдельный cadence примерно раз в восемь кадров;
- накопленное время и число отложенных вызовов ограничены, после чего обновление выполняется;
- текущая location, interaction market и player-owned рынки всегда остаются full-rate;
- отдельный рынок может потребовать full-rate через memory key
  `$starsectorPrepatcher_fullRateMarket=true`.

Scheduler хранит состояние по identity экземпляра рынка, а не через `equals/hashCode`, и очищает
его на границе campaign generation. Вложенный вызов `Economy.advance()` уходит в немедленный
vanilla path, не повреждая контекст внешнего прохода. Перед обычным сохранением pending time
принудительно применяется; transient state не попадает в save graph.

Это намеренная performance-first оптимизация. Суммарный переданный `amount` сохраняется, однако
уменьшение числа вызовов может изменить RNG sequence, точный межрыночный порядок наблюдений и
frame-count callbacks условий, индустрий и submarket-реализаций модов. Для консервативного cadence
нужно использовать `profiles/safe.properties`, выключить `patch.remoteMarketScheduler` целиком или
исключить конкретный рынок через memory key.

## Наблюдение прямых вызовов модов

`patch.directMarketObservation` — отдельный observation-only transformer. Он оборачивает direct call
`MarketAPI.advance(float)` в загружаемом bytecode модов, но не откладывает, не объединяет и не
подавляет его. Receiver, `amount`, текущий thread, порядок, синхронный post-call contract и
распространение исключений сохраняются.

Для каждого найденного call site фиксируются source/JAR, class, method, descriptor, line, ordinal и
best-effort классификация источника `amount`. Runtime наблюдение собирает sampled inclusive timing,
stack signatures, приблизительное число рынков и счётчики recursion, errors и переданного amount.
Entry probe в vanilla `Market.advance()` отдельно учитывает ограниченную выборку reflective,
MethodHandle и иных неинструментированных входов.

Каждый запуск создаёт отдельный каталог `logs/direct-market-observe/session-*` с файлами:

- `call-sites.csv`;
- `observations.csv`;
- `stacks.csv`;
- `unknown-stacks.csv`;
- `summary.csv`;
- `session.json`.

Default/aggressive profile включает observer в этом выпуске, чтобы получить данные для следующих
итераций экономических оптимизаций. Измерение времени по умолчанию выполняется примерно для одного
вызова из 128, stack capture — на первом и затем примерно одном вызове из 2048. После сбора нужных
данных `patch.directMarketObservation` можно выключить, чтобы полностью убрать sampling overhead.
Safe profile оставляет observer выключенным.

## Совместимость и fail-open

Structural matcher scheduler принимает только единственный подтверждённый market call центрального
loop и отдельно проверяет save boundary. Observation transformer ограничен совместимыми game/mod
classloader и исключает core/API/common/sound JAR, а также сам Prepatcher. Неизвестная,
неоднозначная или частично изменённая форма остаётся нетронутой и получает локальный
`SKIPPED_STRUCTURAL`, `SKIPPED_LOADER` или `SKIPPED_ERROR` без отключения остальных патчей.

Observer может быть заметен коду, который проверяет собственные class bytes или анализирует stack:
у обёрнутого вызова появляется дополнительный frame, а class bytes меняются до определения JVM.
Такие случаи не подавляются скрытно и должны разбираться по диагностическому логу.

## Проверка выпуска

Полный release gate пересобирает agent/bootstrap JAR и проверяет:

- documentation/version/link/checksum consistency;
- structural, negative и idempotency contracts всех патчей;
- отдельную трансформацию direct market call sites;
- cadence, hidden-market policy, exact amount conservation, hot policies, identity state,
  reentrancy и save flush scheduler;
- multiplicity, delivered amount, exception propagation, mod-loader transformation и CSV output
  observer;
- lifecycle/GC, exp6, exp8, loading/save и hyperspace regression suites;
- javaagent startup и реальную Faster Rendering classloader topology с 55 runtime payload classes.

Полный список пользовательских изменений находится в [`CHANGELOG.md`](../../CHANGELOG.md), правила
совместимости — в [`COMPATIBILITY.md`](../COMPATIBILITY.md), а обязательная игровая матрица — в
[`VALIDATION.md`](../VALIDATION.md).
